## 📖 项目简介

UE4内存读取器是一款专为Unreal Engine 4/5游戏设计的高性能内存分析工具。本工具采用先进的进程内存读取技术，能够在不暂停目标进程的情况下，安全、高效地读取游戏内存数据。支持自动检测UE4模块基址、智能偏移定位、链式指针追踪等高级功能，是游戏逆向分析和内存监控的理想解决方案。

## ✨ 核心特性

### 🔍 智能地址定位
- **自动基址检测** - 智能扫描并定位UE4模块基地址
- **偏移自动匹配** - 支持多种UE版本偏移模式自动适配
- **GName/GWorld/GObjects检测** - 自动识别UE核心数据结构

### 🛠️ 强大读取能力
- **链式指针追踪** - 支持多级指针链式读取
- **多种数据类型** - 支持指针、浮点数、三维向量等数据读取
- **进程无感读取** - 使用process_vm_readv，不干扰目标进程运行

### 💻 终端交互模式
- **实时命令执行** - 支持动态内存读取和分析
- **地址保存管理** - 可保存常用地址便于快速访问
- **友好交互界面** - 彩色输出和详细状态提示

## 🚀 快速开始

### 系统要求
- Linux 内核 4.9+
- 需要root权限或ptrace能力
- 支持x86_64、ARM64架构

### 编译安装
```bash
# 克隆项目
git https://github.com/Aniansoda/UE_Dump.git
cd UE_Dump

# 编译程序
clang++ -std=c++20 -O3 UE引擎Gworld_Gnane.cpp -o UE_Dump_Gdata

g++ -std=c++20 -O3 UE引擎Gworld_Gnane.cpp -o UE_Dump_Gdata

# 授予执行权限
chmod +x UE_Dump_Gdata
```

基本使用

```bash
# 交互模式（推荐）
sudo ./UE_Dump_Gdata <目标进程PID>

# 单命令模式
sudo ./UE_Dump_Gdata <PID> base              # 显示基础信息
sudo ./UE_Dump_Gdata <PID> game              # 显示游戏地址
sudo ./UE_Dump_Gdata <PID> chain libue4+0x12345678->0x20->0x30  # 链式读取
```

📋 命令手册

🔧 核心命令

命令 功能描述 示例
base 显示UE4基础地址信息 base
game 显示游戏运行时地址 game
player 显示玩家相关信息 player
chain <表达式> 链式指针读取 chain libue4+0x12345678->0x20
read <地址> 读取指定内存地址 read 0x7ff812345678
save <名称> <地址> 保存地址到缓存 save Matrix 0x7ff812345678
saved 显示已保存的地址 saved
update 更新所有动态地址 update
help 显示命令帮助 help

🔗 链式表达式语法

支持复杂的指针链式追踪：

```
libue4+0x12345678->0x20->0x30
[基址]+[偏移]->[下一级偏移]->[...]
```

🛡️ 技术特性

高性能读取

· 双读取引擎 - 优先使用process_vm_readv系统调用，失败时自动回退到/proc/pid/mem
· 智能分块 - 大内存读取自动分块，避免系统资源耗尽
· 错误恢复 - 读取失败自动重试和错误报告

安全可靠

· 进程保护 - 只读操作，不修改目标进程内存
· 权限验证 - 自动检测读取权限和系统兼容性
· 边界检查 - 所有内存访问都经过有效性验证
